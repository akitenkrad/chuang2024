# CLI

Rust バイナリは `chuang` (`cargo run --release -- …` で実行)．サブコマンドは `run` と `sweep` の 2 つ．`reproduce` (論文 Table 1 / Fig. 4–6) は Phase 3 に保留．

## LLM 環境変数

すべての LLM 呼び出しは **Ollama 第一 → OpenAI フォールバック** クライアントを通る．プロバイダは環境変数で設定する (ハードコードしない):

| 変数 | 既定 | 意味 |
|---|---|---|
| `OLLAMA_HOST` | `http://localhost:11434` | Ollama サーバ URL (`/api/chat`)． |
| `OLLAMA_MODEL` | `llama3.1` (socsim-llm 既定) | Ollama モデル．本プロジェクトのクイックスタートは `llama3.2:latest`． |
| `OPENAI_API_KEY` | (未設定) | OpenAI キー．未設定ならフォールバックは no-op プレースホルダ (Ollama も失敗した場合のみ問題)． |
| `OPENAI_MODEL` | `gpt-4o-mini` | フォールバック時の OpenAI モデル． |

```bash
export OLLAMA_HOST=http://localhost:11434
export OLLAMA_MODEL=llama3.2:latest
```

## `run` — 単一設定

```bash
cargo run --release -- run \
    --n-agents 10 --topic flat_earth --framing false --bias none \
    --memory cumulative --topology full \
    --events-per-step 1 --max-steps 100 --tol 1e-6 --seed 42 \
    --temperature 0 --llm-seed 0 --cache-path .llm_cache/cache.json
```

| フラグ | 既定 | 意味 |
|---|---|---|
| `--n-agents` | `10` | エージェント数 N |
| `--topic` | `flat_earth` | 議論トピック (ground truth 既知; アンダースコア可) |
| `--framing` | `false` | 主張の `true` / `false` フレーミング |
| `--bias` | `none` | 確証バイアス: `none` / `weak` / `strong` |
| `--memory` | `cumulative` | メモリ方式: `cumulative` / `reflective` (reflective は Phase 3 スタブ) |
| `--topology` | `full` | `full` (完全グラフ) / `ws` (Watts–Strogatz) / `ba` (Barabási–Albert) |
| `--ws-k` | `4` | WS 初期次数 k (偶数) |
| `--ws-beta` | `0.1` | WS 再配線確率 β |
| `--ba-m` | `2` | BA 1 ノードあたり結合数 m |
| `--events-per-step` | `1` | 1 tick あたりの dyadic interaction 数 (バッチ調整) |
| `--max-steps` | `100` | 最大ステップ T |
| `--tol` | `1e-6` | 意見分散の収束しきい値 |
| `--seed` | ランダム | コア RNG シード (ネットワーク + ペア選択; 決定論的コア層) |
| `--temperature` | `0.0` | LLM 生成温度 (論文は 0.7; 再現性のため既定 0) |
| `--llm-seed` | `0` | LLM 生成シード (Ollama は honour, OpenAI は best-effort) |
| `--cache-path` | `.llm_cache/cache.json` | プロンプト→応答キャッシュファイル (gitignore 対象) |
| `--output-dir` | `results` | 出力ベースディレクトリ |

出力は `results/{timestamp}/` に書かれ，`results/latest` が貼り直される:

- `config.json` — 実行設定一式．
- `opinions.csv` — long-format `t, agent_id, opinion[, text]`．`text` 列は最終ステップのみ各エージェントの最終ツイートを格納する (途中ツイートはファイル肥大化を避けて省略)．
- `metrics.csv` — `t, variance, bias, diversity, n_clusters, polarization`．
- `run_metadata.json` — LLM モデル / endpoint / 温度 / seed / 呼び出し総数 / cache-hit / cache-hit 率 と決定論注記．

## `sweep` — 確証バイアス × フレーミング × トポロジ

```bash
cargo run --release -- sweep \
    --bias-values none,weak,strong \
    --framing-values true,false \
    --topology-values full,ws \
    --memory cumulative --n-agents 10 --runs 5 \
    --max-steps 100 --seed 42 \
    --cache-path .llm_cache/cache.json
```

| フラグ | 既定 | 意味 |
|---|---|---|
| `--bias-values` | `none,weak,strong` | カンマ区切りの確証バイアス格子 |
| `--framing-values` | `true,false` | カンマ区切りのフレーミング格子 |
| `--topology-values` | `full` | カンマ区切りのトポロジ格子 |
| `--memory` | `cumulative` | sweep 全体で単一のメモリ方式 |
| `--topic` | `flat_earth` | 議論トピック |
| `--n-agents` | `10` | エージェント数 N |
| `--runs` | `5` | 各条件あたりの独立試行数 (各々独立な派生シード) |
| `--events-per-step` | `1` | 1 tick あたりの interaction 数 |
| `--max-steps` | `100` | 最大ステップ T |
| `--tol` | `1e-6` | 収束しきい値 |
| `--seed` | `42` | 基点シード (各条件/試行は独立ストリームを派生) |
| `--temperature` / `--llm-seed` | `0` / `0` | LLM 生成パラメータ |
| `--cache-path` | `.llm_cache/cache.json` | sweep 全体で共有するキャッシュ (ヒット率を上げる) |
| `--output-dir` | `results` | 出力ベースディレクトリ |

出力は `results/{timestamp}_sweep/` に:

- `sweep_summary.csv` — `(bias, framing, topology, run)` ごとに `final_bias` / `final_diversity` / `final_variance` / `n_clusters` / `polarization` / `convergence_time` / `converged` / `cache_hit_rate` を 1 行．
- `sweep_config.json` — sweep 格子と共通パラメータ．

コンソールは確証バイアス別の平均 Diversity `D̄` を表示する．論文の核心は `D̄` が `none → weak → strong` で単調増大することである．

> **コスト注意.** 各ステップは LLM を 2 回 (話者 + 聴者) 呼び，時折分類器フォールバックが加わる．sweep は それ × 条件数 × 試行数 × ステップ数 になる．sweep 全体でキャッシュを共有し，`--n-agents` / `--max-steps` / `--runs` を小さく始めること．
